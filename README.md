## 🧩 各环节详细说明（含优化措施）

### 1. 系列管理

- **存储结构**：`workspace/{series_name}/` 作为唯一命名空间。
- **功能**：创建、切换、删除系列。每个系列完全独立，含自己的剧本、角色、场景、输出。
- **数据隔离**：通过文件夹物理隔离，无数据库，所有元数据存于各文件夹下 `manifest.json`。

### 2. 📝 剧本工程

| 项目 | 内容 |
|:---|:---|
| **API** | DeepSeek-V4 |
| **输入** | 用户自由文本（单幕或多幕/多集剧本） |
| **输出** | 结构化JSON（见下方schema），包含所有场景、分镜、对话、标注 |
| **关键能力** | 自动拆解场景、识别角色/情绪/镜头语言，支持“第X集”嵌套 |
| **优化点** | 1. 内置**安全词映射表**（如“血迹”→“红色液体”），在生成分镜描述时自动过滤敏感词。<br>2. 强制遵循 JSON Schema，出错时自动重试。<br>3. 输出的角色列表和场景列表直接驱动后续环节。 |

**标准剧本JSON结构**：

```json
{
  "series": "暗夜追踪",
  "episode": 1,
  "scenes": [
    {
      "scene_id": 1,
      "location": "废弃仓库",
      "time": "深夜",
      "shots": [
        {
          "shot_id": 1,
          "description": "全景，男主角从阴影中走出，手电筒光柱扫过堆积的货箱。",
          "camera": { "angle": "平视", "movement": "跟" },
          "dialogues": [],
          "characters": ["男主"],
          "emotion": "警觉"
        }
      ]
    }
  ]
}
```

### 3. 🎭 角色工程（角色一致性系统）

| 项目 | 内容 |
|:---|:---|
| **API** | GPT-IMAGE 2 |
| **输入** | 剧本JSON中的角色列表 |
| **输出** | ① 角色圣经JSON（6层锚点）<br>② 正面/侧面/背面三视图<br>③ 特征分解九宫格图<br>④ 表情集（可选） |
| **一致性优化** | **三级保障链**：<br>**L1 参考图增强**：九宫格图作为最高优先级参考。<br>**L2 动态提示词注入**：视频生成时自动将关键特征用双括号强调，如 `((男，银色短发，左眼泪痣，锈红皮风衣))`。<br>**L3 自动校验闭环**：视频生成后，用CLIP比对首帧与角色参考图；若相似度<0.85，自动截图为首帧并追加提示词“严格按照此图生成”，重新请求API。 |
| **重新生成** | 前端可修改任何锚点描述，触发单张或全部参考图重绘。 |

### 4. 🌄 场景工程

| 项目 | 内容 |
|:---|:---|
| **API** | GPT-IMAGE 2（与角色一致，风格统一） |
| **输入** | 剧本JSON中的场景列表，每个场景有文字描述。 |
| **输出** | ① 结构化场景描述JSON（光照、色调、时代、地理）<br>② 多视角图集：定场镜头(wide)、角落细节(close-up)、俯视图(bird-eye) |
| **优化点** | 1. **全局风格板注入**：所有场景图生成时附加统一的风格板提示（如“写实、低饱和、暖色基调”），确保系列视觉一致。<br>2. 场景描述自动转为视觉提示词（通过预置模板）。 |
| **重新生成** | 同角色，支持修正描述后重绘。 |

### 5. 🎬 导演配置（分镜精控）

| 类别 | 可配置项（按钮/下拉） | 说明 |
|:---|:---|:---|
| **画幅** | 横屏16:9、竖屏9:16、方形1:1、宽银幕21:9 | 决定输出比例 |
| **风格** | 2D动画、3D卡通、写实、赛博朋克、水墨、定格、皮克斯、日漫…… | 统一视觉基调 |
| **质量** | 480p(草稿)、720p(预演)、1080p(成品)、2K/4K(最高) | 成本分级 |
| **景别** | 大远景、全景、中景、近景、特写、大特写 | 镜头范围 |
| **机位角度** | 平视、俯视(鸟瞰)、仰视、荷兰角、过肩、POV | 视角选择 |
| **运镜** | 固定、推、拉、摇、移、跟、升降、手持晃动 | 运动方式 |
| **光影** | 自然光、黄金时刻、阴天、霓虹灯、顶光、侧逆光、剪影 | 氛围光效 |
| **色彩** | 暖色、冷色、高饱和、低饱和、黑白、复古胶片 | 色调LUT |
| **节奏** | 缓慢抒情、正常叙事、快速紧张、MV跳跃 | 剪辑暗示 |
| **时长** | 2s / 4s / 6s / 8s / 10s / 15s | 单镜头长度 |
| **音频参考** | 上传音频文件（可选），用于节奏控制或数字人驱动 | 仅限支持多模态的模型 |

**导演卡存储**：每个分镜的配置存入`storyboard.json`，与剧本镜头一一对应。

### 6. ⭐ 视频生成（最终输出）

| 项目 | 内容 |
|:---|:---|
| **主选API** | Seedance 2.0（火山引擎/即梦） |
| **备选API** | 通义万相 Wan2.6（阿里模型工作室）<br>本地兜底：CogVideoX / AnimateDiff（仅限极端情况） |
| **输入拼装** | 从当前系列文件夹聚合：<br>① 分镜剧本描述<br>② 角色圣经（文本）+ 所有参考图<br>③ 场景图（选最匹配视角）<br>④ 导演卡配置<br>⑤ 可选音频参考 |
| **异步架构** | 1. 后端提交任务，立即返回 `task_id`。<br>2. 任务状态持久化到本地 `tasks.json`。<br>3. 前端轮询或通过 WebSocket 显示进度。<br>4. 失败自动重试（指数退避），若安全拦截则自动改写提示词并重试。 |
| **多镜头处理** | - **镜头包策略**：连续2-3个镜头打包，共用参考图，减少跳跃。<br>- **过渡空镜**：在镜头间自动插入1秒“环境空镜”（如树叶、路灯），掩盖衔接瑕疵。 |
| **后处理** | - 所有生成的镜头用 `ffmpeg` 进行直方图匹配/色彩统一。<br>- 根据导演卡拼接成完整场景视频。 |
| **下载** | 提供单镜头、单场景、完整集三个粒度的下载，存储在本地`outputs/`。 |

### 7. 系统级保障与运维

- **内存索引**：启动时扫描所有系列`manifest.json`构建索引缓存，用`watchdog`实时更新，支持快速模糊搜索。
- **速率控制**：任务队列（RQ/Celery）搭配`rate_limit`，自动处理429限流。
- **成本熔断**：按日/系列设置费用上限，超限后暂停提交并通知。
- **多模型抽象**：通过统一的`VideoGenerator`接口类，实现多厂商无缝切换。
- **垃圾回收**：定期清理`outputs/`中超过30天且未标记“存档”的中间视频。
- **断点续传**：服务重启后，根据`tasks.json`继续查询远端任务状态。

## ✅ 最终工作流执行示例（宏指令）

```
用户操作: 选择系列“暗夜追踪” → 输入完整剧本 → 点击“一键处理”
系统自动执行:
1. DeepSeek-V4 结构化剧本 (30s)
2. 提取角色列表 → GPT-IMAGE 2 生成角色圣经+三视图+九宫格 (2min)
3. 提取场景列表 → GPT-IMAGE 2 生成多视角场景图 (1min)
4. 用户审核并微调角色/场景图 (人工)
5. 用户进入导演模式，为每个分镜点选配置 (5min)
6. 点击“生成全部” → 后端按镜头包分批提交 Seedance 2.0 → 异步队列处理 (5-20min)
7. 自动校验一致性，不合格重生成
8. 色彩匹配 → 插入过渡空镜 → 合成完整视频 → 提供下载
```
