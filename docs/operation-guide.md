# 本地 AI 视频工作流操作说明

本文档面向日常使用，不讲实现细节，只讲怎么启动、怎么生成、文件会落到哪里。

## 1. 项目结构

项目固定结构如下：

```text
output/              所有本地生成数据
start.bat            启动前后端
restart.bat          重启前后端
stop.bat             停止前后端
code/
├── frontEnd/        前端
└── backEnd/         后端
```

所有业务数据都写入 `output/`，不使用数据库。

## 2. 启动方式

首次启动前请确认：

- 已安装 Python 3.11+
- 已安装 Node.js 18+
- `code/backEnd/.env` 中已配置可用的 API Key

常用命令：

1. 双击 `start.bat`
2. 日常重启使用 `restart.bat`
3. 停止服务使用 `stop.bat`

默认访问地址：

- 前端：`http://127.0.0.1:8080`
- 后端：`http://127.0.0.1:8000`

## 3. 基本使用流程

### 3.1 创建系列

先在左侧创建一个系列。

系列是最高级隔离单位。后续的剧本、角色、场景、分镜、快照、任务都会存到该系列目录下。

系列支持直接编辑和删除：

- 修改系列名称或简介后，点击“保存修改”
- 点击“删除系列”会直接删除该系列在 `output/` 下的整套本地数据

### 3.2 创建剧集并录入剧本

在当前系列下：

1. 新建剧集
2. 在“剧本输入区”粘贴原始剧本
3. 点击“AI 拆解剧本”
4. 检查右侧 JSON 结果
5. 如有需要，手动修改后点“保存 JSON”

剧本解析走 DeepSeek。

剧集支持直接编辑和删除：

- 修改当前剧集名称后，点击“保存剧集”
- 删除剧集前，必须先清掉引用它的场景和分镜板，否则系统会阻止删除

### 3.3 创建角色

在“角色创建”区域：

1. 输入角色名
2. 输入角色简介、身份、性格、造型说明
3. 点击“新建角色”

创建后，角色默认是草稿状态。

角色支持直接编辑和删除：

- 修改角色名称或简介后，点击“保存角色”
- 删除角色前，必须先移除所有引用该角色的镜头，否则系统会阻止删除

### 3.4 上传角色参考图

如果是固定角色，建议先上传参考图。

适用场景：

- 已有官图、设定图、剧照
- 想让生成结果尽量贴近既有角色

上传后，参考图会保存到：

`output/<series>/characters/<character_id>/source_uploads/`

每张已上传参考图都支持单独删除。

### 3.5 生成角色圣经

角色支持两种生成模式：

#### 模式 A：参考图 + 文字生成

适合：

- 有参考图，但仍希望结合你填写的角色文字描述
- 需要保留剧情设定、性格和道具信息

效果：

- 会综合剧本上下文、角色简介、上传参考图一起生成

#### 模式 B：仅按参考图生成拼图

适合：

- Tom、Jerry 这类已有明确视觉主体的角色
- 只想让模型按上传图做三视图和特征拆解

效果：

- 上传图作为唯一主体来源
- 不再把文字描述混入图片主体重设计
- 输出单张角色圣经拼图

角色圣经输出内容：

- 三视图：正面、侧面、背面
- 特征分解九宫格：面部、头部细节、服装细节、饰品、标志性姿势等

输出文件位置：

`output/<series>/characters/<character_id>/refs/character_bible_sheet.jpg`

### 3.6 创建场景

在“场景创建”区域：

1. 输入场景名
2. 输入场景描述
3. 点击“新建场景”

场景支持直接编辑和删除：

- 修改场景名称或描述后，点击“保存场景”
- 删除场景前，必须先移除所有引用该场景的镜头，否则系统会阻止删除

### 3.7 生成场景包和场景参考拼图

点击“生成场景参考拼图”后，系统会：

1. 用 DeepSeek 生成场景结构化描述
2. 提炼环境视觉信息
3. 生成单张场景参考拼图

场景拼图包含：

- 定场大景
- 氛围近景
- 鸟瞰或高角度空间关系
- 关键细节或道具区域

输出文件位置：

`output/<series>/scenes/<scene_id>/refs/scene_reference_sheet.jpg`

### 3.8 创建分镜和镜头配置

在右侧分镜区：

1. 新建分镜板
2. 选择关联场景
3. 选择镜头景别
4. 选择运镜方式
5. 填写光线和色调关键词
6. 选择出镜角色
7. 创建镜头卡

当前还支持：

- 删除分镜板
- 行内编辑镜头的场景、景别、运镜、时长、光线、色调、角色
- 删除镜头

当前前端可配置的核心项包括：

- 场景
- 景别
- 运镜
- 光线
- 色调
- 时长
- 角色

删除限制：

- 分镜板如果已经被快照引用，则不能删除
- 镜头如果已经被快照或任务引用，则不能删除

### 3.9 组装镜头包

点击“组装镜头包”后，系统会把以下内容整理到一个统一提示包：

- 剧本上下文
- 角色锚点
- 角色参考图
- 角色上传原图
- 场景场景包
- 场景参考拼图
- 镜头配置

这一步是为后续视频模型接入做准备。

### 3.10 任务草稿

当前视频模型还未最终接入，因此目前可以先测试到：

- 剧本拆解
- 角色生成
- 场景生成
- 镜头包组装
- 任务草稿落盘

## 4. 删除规则

为了避免本地数据出现悬挂引用，当前删除逻辑有以下限制：

- 系列：直接整包删除
- 剧集：若仍被场景或分镜板引用，则不能删除
- 角色：若仍被镜头引用，则不能删除
- 场景：若仍被镜头引用，则不能删除
- 角色参考图：可单独删除，不影响角色主体数据

### 4.1 角色

```text
output/<series>/characters/<character_id>/
├── character.json
├── bible.json
├── refs/
│   └── character_bible_sheet.jpg

└── generated/
```

## 5. 常见问题

### 5.1 生成成功但前端没图

优先检查：

1. 前端是否已重启
2. 图片文件是否已实际写入 `output/`
3. 当前系列是否选择正确

### 5.2 上传参考图后无法上传

正常流程是：

1. 先选择角色
2. 再选择本地图片文件
3. 再点击“上传参考图”

### 5.3 角色和场景图太大

当前默认已做轻量化：

- 角色图：`1024x1536`，`jpeg`
- 场景图：`1536x1024`，`jpeg`

如需调整，可修改：

- `GPT_IMAGE_CHARACTER_SHEET_SIZE`
- `GPT_IMAGE_CHARACTER_QUALITY`
- `GPT_IMAGE_OUTPUT_COMPRESSION`
- `SCENE_IMAGE_SHEET_SIZE`
- `SCENE_IMAGE_QUALITY`
- `SCENE_IMAGE_OUTPUT_COMPRESSION`
