# 本地 AI 视频工作流使用说明

本文档面向日常使用，按当前页面和代码实现整理，不讲内部实现细节，只讲怎么操作。

## 1. 先理解页面

当前界面按 4 列使用最清晰：

1. 第 1 列：系列、剧集、原始剧本、解析结果
2. 第 2 列：角色、场景、角色圣经、场景参考图
3. 第 3 列：分镜板、镜头卡、生产模式、生成参数
4. 第 4 列：执行区
   内容包括镜头包/场景包、快照、提交草稿、远端任务

使用原则：

- 前 3 列负责准备数据和配置
- 第 4 列负责执行、提交、看结果

## 2. 启动方式

### 2.1 环境要求

- Python 3.11+
- Node.js 18+
- `code/backEnd/.env` 已配置可用的 API Key

### 2.2 常用脚本

- `start.bat`
- `restart.bat`
- `stop.bat`

### 2.3 默认地址

- 前端：`http://127.0.0.1:8080`
- 后端：`http://127.0.0.1:8000`

## 3. 标准使用流程

推荐按下面顺序走，不容易乱。

### 3.1 创建系列

先在第 1 列创建一个系列。

系列是最高层的工作区。后续的剧本、角色、场景、分镜板、快照、任务都会保存到该系列目录下。

### 3.2 创建剧集并录入原始剧本

在当前系列下：

1. 新建剧集
2. 粘贴原始剧本
3. 点击“AI 拆解剧本”
4. 查看解析结果

解析结果支持两种阅读方式：

- 可读视图：适合普通用户直接看和改
- 原生结构：适合查看 JSON 原始结构

### 3.3 修改解析后的场景和分镜

如果 AI 解析结果不准确，不建议直接手改 JSON 字符串。

推荐在可读视图里直接修改：

- 场景
- 分镜
- 对白
- 情绪 / 节拍
- 机位角度 / 运镜方式 / 景别

同时支持：

- “导入为镜头卡草稿”
- “一键导入全部为镜头卡草稿”

## 4. 角色和场景资产

### 4.1 创建角色

在第 2 列创建角色，填写：

- 角色名
- 角色简介
- 身份、性格、外观说明

如果是固定角色，建议上传参考图。

### 4.2 生成角色圣经拼图

角色支持两种生成方式：

- `参考图 + 文字生成`
- `仅按参考图生成拼图`

当前实际推荐使用方式：

- 有官方图、设定图、参考图时，优先上传
- 最终用于视频提示词的角色参考图，优先使用“角色圣经拼图”

常见输出：

- 正面 / 侧面 / 背面
- 五官、发型、服装等拆解区域

### 4.3 创建场景

在第 2 列创建场景，填写：

- 场景名
- 场景描述
- 环境、氛围、时间、空间信息

### 4.4 生成场景参考拼图

点击“生成场景参考拼图”后，系统会生成场景的参考图集合。

当前实际推荐使用方式：

- 视频提示词里的场景参考图，优先使用“场景参考拼图”

## 5. 分镜板与生产模式

### 5.1 创建分镜板

在第 3 列点击“新建分镜板”。

### 5.2 两种生产模式

分镜板支持两种模式：

#### 分镜生产

适合：

- 一镜一任务
- 每个镜头独立生成
- 后期再拼接成整段视频

优点：

- 控制更细
- 更容易逐镜头修正

#### 场景直出

适合：

- 一整个场景一次生成
- 不按单镜头分别提交

优点：

- 操作更短
- 适合快速验证整段剧情

如果你不确定选哪个，优先选 `分镜生产`。

## 6. 分镜生产模式怎么用

### 6.1 创建镜头卡

镜头卡可以来自两种入口：

- 手动新建镜头卡
- 从解析剧本导入镜头卡草稿

### 6.2 每张镜头卡需要检查什么

重点字段：

- 关联场景
- 输入模式
- 比例
- 分辨率
- 时长
- 生成数量
- 是否有声音
- 景别
- 运镜方式
- 角色
- 剧情描述
- 情绪
- 节拍
- 对白
- 剧本原文摘录

### 6.3 输入模式

当前镜头卡支持 3 种输入模式：

#### 纯文字生成

- 只使用文字描述
- 不显示参考图上传
- 不使用首尾帧

#### 参考生成

- 使用剧本文字描述
- 自动带入角色圣经拼图
- 自动带入场景参考拼图
- 不显示首尾帧上传

#### 首尾帧生成

- 使用剧本文字描述
- 使用首帧 / 尾帧图片
- 首帧必传，尾帧可选
- 不使用角色圣经拼图和场景参考拼图

## 7. 场景直出模式怎么用

场景直出模式不按单镜头推进，而是按一个场景推进。

操作步骤：

1. 分镜板切到 `场景直出`
2. 选择场景
3. 配置输入模式、比例、分辨率、时长、声音等
4. 去第 4 列执行区推进

如果是参考生成，系统会按当前场景和角色配置自动组织参考图。

## 8. 第 4 列执行区怎么理解

这是当前最重要的区域。

执行区统一分成 3 步：

1. 镜头包 / 场景包
2. 快照
3. 提交草稿

之后才是：

4. 提交任务
5. 刷新状态
6. 查看远端任务和结果

### 8.1 镜头包 / 场景包

这是“生成语义材料”的组装结果。

会把这些内容整理到一起：

- 当前镜头或场景的剧情信息
- 剧本原文摘录
- 当前镜头台词片段
- 角色参考图
- 场景参考图
- 视觉参数
- 生成规格

注意：

- 镜头包不是最终请求体
- 它更像“提交前的提示词和素材组装结果”

### 8.2 快照

快照是对当前组装结果的冻结。

作用：

- 留档
- 回溯
- 给后续提交草稿使用固定版本

你可以理解成：

- 镜头包是当前工作结果
- 快照是正式落盘版本

### 8.3 提交草稿

提交草稿会基于快照，生成最终要发给 Seedance 的 `request_body`。

这里会看到两种展示：

- `Seedance 提交草稿`
- `Seedance 最终提交体`

区别：

- 提交草稿：还没真正发给官方前的本地 request body
- 最终提交体：真正提交时已转换成最终版本，例如图片已 materialize 成 base64

## 9. 单镜头任务怎么走

单镜头完整流程：

1. 在第 3 列选中一张镜头卡
2. 去第 4 列点击“生成镜头包”
3. 生成快照
4. 点击“生成提交草稿”
5. 检查 `request_body`
6. 点击“提交任务”
7. 点击“刷新状态”
8. 查看任务结果

## 10. 批量镜头任务怎么走

如果你已经有多张镜头卡，例如 5 张：

1. 勾选多个镜头卡
2. 点击“生成选中镜头草稿”或“生成全部镜头草稿”
3. 系统会创建一个批次
4. 每个镜头会各自生成：
   - 镜头包
   - 快照
   - 本地任务草稿

### 10.1 批次区常用按钮

- `提交失败/未提交草稿`
- `批量刷新当前批次状态`
- `重试失败镜头`

### 10.2 要特别注意

批量镜头任务 = 多条独立视频任务。

例如 5 个镜头，就是 5 条独立任务，不是一条长视频。

如果你的目标是一整段一次生成，应该改用 `场景直出模式`。

## 11. 本地任务和远端任务的区别

### 11.1 本地任务草稿

本地 `job` 负责：

- 记录 snapshot_id
- 保存 request body
- 保存本地状态
- 保存结果路径

### 11.2 远端任务

远端任务是 Seedance / Ark 那边的执行记录，主要包含：

- task_id
- 远端状态
- 官方返回体

推荐理解方式：

- 本地任务草稿：你的生产记录
- 远端任务：平台执行记录

日常以本地任务为主，排错时再看远端任务。

## 12. 当前推荐操作顺序

如果你现在只是想稳定出一轮内容，建议按下面顺序使用：

1. 选系列
2. 选剧集
3. 录入原始剧本
4. AI 拆解剧本
5. 在可读视图里修正场景和分镜
6. 创建角色并生成角色圣经拼图
7. 创建场景并生成场景参考拼图
8. 创建分镜板，选择 `分镜生产`
9. 一键导入全部为镜头卡草稿
10. 逐张检查镜头卡
11. 先试跑 1 张镜头：
    - 生成镜头包
    - 生成提交草稿
    - 检查 request body
12. 没问题后，再批量生成全部镜头草稿
13. 最后统一提交

## 13. 删除规则

为了避免本地数据悬挂引用，当前删除不是完全自由的。

### 13.1 系列

- 删除系列会直接删除该系列目录下的本地数据

### 13.2 剧集

- 如果剧集仍被场景或分镜板引用，则不能删除

### 13.3 角色

- 如果角色仍被镜头引用，则不能删除

### 13.4 场景

- 如果场景仍被镜头引用，则不能删除

### 13.5 分镜板

- 如果分镜板仍被快照或任务引用，则不能删除

### 13.6 镜头

- 如果镜头仍被快照或任务引用，则不能删除

### 13.7 快照

- 如果快照仍被任务引用，则不能删除

### 13.8 任务草稿

- 如果任务已经进入远端执行流程，不能直接删除
- 需要先取消远端任务，或保留记录

## 14. 常见问题

### 14.1 现在能直接生成最终视频吗

可以走到提交官方任务这一步，但前提是你的账号已开通：

- `doubao-seedance-2-0-260128`

如果模型未开通，本地流程仍可正常使用：

- 镜头包 / 场景包
- 快照
- 提交草稿

但真正提交远端时会失败。

### 14.2 为什么我看到了很多“草稿”

因为当前流程是显式分层的：

- 镜头卡 / 场景配置
- 镜头包 / 场景包
- 快照
- 提交草稿
- 远端任务

这样做的好处是：

- 可回溯
- 可重试
- 可批量
- 不容易因为一步失败丢上下文

### 14.3 为什么建议优先用分镜生产

因为它更稳定、更可控：

- 每个镜头可单独调
- 失败可重试单镜头
- 更适合后期拼接

### 14.4 一整段剧情怎么一次生成

使用 `场景直出模式`。

### 14.5 批量 5 个镜头会不会变成一条视频

不会。

批量镜头模式会生成 5 条独立任务。
